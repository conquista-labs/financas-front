# new-domain-model

Cria arquivos de model tipados em `src/domain/models`. Esses são os contratos de
request/response/DTO que o slice do `new-usecase` importa de `@/domain/models`. Os models espelham
o schema OpenAPI do backend.

## Regra crítica — NÃO invente campos

O backend é NestJS + Prisma; os models aqui são **gerados a partir do `swagger.json` real**
(cada arquivo carrega o cabeçalho "auto generated by OpenAPI Generator … Do not edit manually").
O formato dos campos é um contrato, não um chute. Antes de criar um model:

1. Consulte `design_handoff_nossa_grana 2/API_REFERENCE.md` para o formato documentado do recurso.
2. Se os campos exatos não estiverem documentados lá, **pergunte ao usuário** (ou consulte o
   swagger) em vez de inventar nomes/tipos de campo. Um model errado quebra a camada de dados de
   forma silenciosa.

## Convenções (bata exatamente com os arquivos existentes)

- **Uma interface por arquivo**, nome do arquivo em camelCase batendo com o tipo
  (`createPessoaRequest.ts` → `export interface CreatePessoaRequest`).
- Mantenha o bloco de comentário/cabeçalho OpenAPI que todo irmão tem (copie verbatim).
- Models de **response** normalmente envelopam o payload:
  `export interface Get<X>Response { message: string; data: <Payload>; }` e importam `<Payload>`
  do seu próprio arquivo de model. Payloads paginados referenciam `PaginationMeta` + um tipo
  `*PaginatedData` (ver `transacaoPaginatedData.ts`).
- Models de **request** listam os campos graváveis, obrigatórios vs. opcionais (`?`) conforme a
  API (ex.: só `nome` obrigatório em `CreatePessoaRequest`). Adicione o JSDoc por campo quando o
  padrão do irmão usar.
- **Enums** dentro de um request usam o padrão `namespace` + `const` (ver
  `createTransacaoRequest.ts`) — mas prefira buscar os valores das opções em runtime via
  `GET /enums` na UI; o tipo do enum aqui é só para o contrato do request.

## Registre no barrel

Adicione `export * from "./<nomeDoArquivo>";` em `src/domain/models/index.ts`, mantendo o
agrupamento/ordem das entradas ao redor.

## Verificação

- **yarn 4 apenas** — nunca `npm`.
- `yarn tsc --noEmit` (ou `yarn build`) para confirmar que os tipos resolvem e o barrel está
  consistente.
- Reporte os arquivos criados e confirme que cada campo veio do API_REFERENCE/swagger, não foi
  inventado.
